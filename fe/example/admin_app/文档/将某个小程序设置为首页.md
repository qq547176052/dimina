# 将某个小程序设置为首页（App 启动直接进入）

## 目标效果

- App 冷启动直接进入指定小程序（如 `admin_app`），不再经过 `MainActivity` 中转。
- 该小程序作为首页（root 页）**不可被关闭**：
  - 点击胶囊「关闭」按钮 → 不关闭，移动到后台（`moveTaskToBack`）。
  - 按系统返回键 → 若当前是首页则移动到后台；若已跳转到二级页则正常 `finish()` 回退。
- 首页隐藏系统导航栏与胶囊（沉浸式自定义导航）。

## 整体方案

方案由三部分组成：

1. **注册默认小程序**：在宿主 `Application` 中通过 `Dimina.setDefaultMiniProgram()` 注册一个 `root = true` 的小程序，作为 LAUNCHER 启动时的兜底入口。
2. **DiminaActivity 作为 LAUNCHER**：把启动 `Activity` 从 `MainActivity` 改为 `DiminaActivity`，并在 `onCreate` 中优先从 `Intent` 取小程序，取不到时回退到默认小程序。
3. **首页不可关闭 + 自定义导航**：通过 `app.json` 的 `navigationStyle` / `capsuleStyle` / `canClose` 配置，控制导航栏、胶囊显隐与关闭行为；`DiminaActivity` 据此拦截返回键与关闭按钮。

## 涉及文件与改动

### 1. 宿主 Application：注册默认小程序

`android/app/src/main/kotlin/com/didi/dimina/demo/App.kt`

在 `onCreate` 中 `Dimina.init(...)` 之后调用：

```kotlin
Dimina.getInstance().setDefaultMiniProgram(
    MiniProgram(
        appId = "wxd58cedf6d1e1c52c",
        name = "admin_app",
        versionCode = 1,
        versionName = "1.0.0",
        root = true,                       // 必须：标记为 root 首页
        path = "page/tabBar/component/index",
    )
)
```

### 2. Dimina：增加默认小程序存储

`android/dimina/src/main/kotlin/com/didi/dimina/Dimina.kt`

```kotlin
private var defaultMiniProgram: MiniProgram? = null

fun setDefaultMiniProgram(miniProgram: MiniProgram) {
    defaultMiniProgram = miniProgram
}

fun getDefaultMiniProgram(): MiniProgram? = defaultMiniProgram
```

### 3. Manifest：DiminaActivity 作为 LAUNCHER

`android/app/src/main/AndroidManifest.xml`

将 `MAIN` / `LAUNCHER` 的 `<intent-filter>` 移到 `DiminaActivity`，并解决与库模块的 `exported` 合并冲突：

```xml
<activity
    android:name="com.didi.dimina.ui.container.DiminaActivity"
    android:exported="true"
    android:theme="@style/Theme.DiminaAndroid"
    tools:replace="android:exported">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>
</activity>
```

> 说明：`dimina` 库模块默认声明 `android:exported="false"`，宿主覆盖为 `true` 时必须加 `tools:replace="android:exported"`，否则 Manifest 合并报错。

### 4. DiminaActivity：兜底取默认小程序 + 拦截返回/关闭

`android/dimina/src/main/kotlin/com/didi/dimina/ui/container/DiminaActivity.kt`

- `onCreate` 解析小程序（LAUNCHER 兜底）：

```kotlin
val program = getMiniProgramFromIntent(intent)
    ?: Dimina.getInstance().getDefaultMiniProgram()
```

- 拦截返回键（首页且不可关闭 → 后台；否则正常回退）：

```kotlin
onBackPressedDispatcher.addCallback(this, object : OnBackPressedCallback(true) {
    override fun handleOnBackPressed() {
        if (!canClose.value && miniProgram.root) {
            moveTaskToBack(true)
        } else {
            finish()
        }
    }
})
```

- 关闭按钮 `closeMiniProgram()` 同样处理：

```kotlin
private fun closeMiniProgram() {
    if (!canClose.value && miniProgram.root) {
        moveTaskToBack(true)
        return
    }
    activityRegistry.closeAll(miniProgram.appId) { activity -> /* ... */ }
}
```

- 样式与开关由合并后的页面配置驱动：

```kotlin
private fun setInitialStyle(config: MergedPageConfig) {
    showNavigationBar.value = config.navigationStyle != "custom"
    showCapsule.value = config.capsuleStyle != "none"
    canClose.value = config.canClose
    // ...
}
```

> 注意：`miniProgram.root` 来自 `MiniProgram.root`；`canClose` 来自合并配置。只有「首页 + 不可关闭」同时满足时，返回/关闭才会移动到后台。

### 5. 配置链路（导航栏/胶囊/可关闭）

字段 `navigationStyle` / `capsuleStyle` / `canClose` 在以下位置透传：

- 数据模型：`AppConfig.kt` 的 `WindowConfig` 与 `PageModule` 各新增 `capsuleStyle`、`canClose`。
- 合并结果：`BridgeOptions.kt` 的 `MergedPageConfig` 新增 `capsuleStyle: String`、`canClose: Boolean`。
- 合并逻辑：`Utils.kt` 的 `mergePageConfig` 优先级为 **页面私有配置 → 全局 window 配置 → 默认值**：

```kotlin
capsuleStyle = pagePrivateConfig.capsuleStyle ?: appWindowConfig.capsuleStyle ?: "default",
canClose     = pagePrivateConfig.canClose     ?: appWindowConfig.canClose     ?: true,
```

## 小程序侧配置（app.json）

`fe/example/admin_app/app.json` 的 `window` 字段：

```json
"window": {
  "navigationStyle": "custom",   // 隐藏系统导航栏（沉浸式）
  "capsuleStyle": "none",        // 隐藏右上角胶囊
  "canClose": false,             // 首页不可关闭
  ...
}
```

字段含义：

| 字段 | 取值 | 含义 |
| --- | --- | --- |
| `navigationStyle` | `custom` | 隐藏默认导航栏，自定义导航 |
| `capsuleStyle` | `none` | 隐藏胶囊；其它值显示 |
| `canClose` | `false` | 首页不可关闭（返回/关闭 → 移至后台） |

> 注意：`navigationStyle` 必须写 `"custom"`，代码判断为 `!= "custom"` 才显示导航栏；早期误写的 `"none"` 无效。

## 验证清单

- [ ] 冷启动 App 直接进入 `admin_app` 首页，无 `MainActivity` 闪现。
- [ ] 首页无系统导航栏、无右上角胶囊。
- [ ] 在首页按返回键 → App 移动到后台（Recent 任务仍在）。
- [ ] 在首页点击关闭按钮 → App 移动到后台。
- [ ] 从首页 `navigateTo` 进入二级页后按返回键 → 正常回退到首页（不进后台）。
- [ ] 二级页返回到首页后再次按返回键 → 进入后台。
